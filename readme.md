# TeamFinder

Приложение для поиска команды в пет-проекты и совместной разработки.

Разработчики, дизайнеры, тестировщики и другие специалисты могут создавать собственные проекты, находить участников и присоединяться к существующим командам.

---

## Возможности

### Пользователи

- Регистрация по email и паролю
- Авторизация и выход из аккаунта
- Просмотр профилей пользователей
- Редактирование профиля:
  - аватар
  - имя
  - фамилия
  - контакты
  - описание
- Смена пароля

### Аватары

Если пользователь не загрузил изображение профиля, автоматически создаётся аватар:

- первая буква имени пользователя;
- случайный цветной фон.

### Проекты

- Создание проектов
- Редактирование собственных проектов
- Завершение проектов
- Просмотр списка проектов
- Просмотр детальной информации о проекте

### Участие в проектах

- Подать заявку на участие в проекте
- Отказаться от участия
- Просматривать участников проекта

### Избранное

- Добавление проектов в избранное
- Удаление из избранного
- Отдельная страница со всеми сохранёнными проектами

### Фильтрация пользователей

После авторизации доступны дополнительные фильтры:

- авторы проектов, добавленных в избранное;
- авторы проектов, в которых пользователь участвует;
- пользователи, добавившие в избранное ваши проекты;
- участники ваших проектов.

### Пагинация

Во всех списках используется постраничная навигация:

- по 12 элементов на страницу.

### Администрирование

Доступна административная панель Django для управления:

- пользователями;
- проектами;
- участниками;
- избранным.

## Установка и запуск

### 1. Клонирование репозитория

```bash
git clone https://github.com/mystbane-byte/team-finder-ad.git
cd team-finder-ad
```

### 2. Создание виртуального окружения

```bash
python -m venv venv
```

#### Windows

```bash
venv\Scripts\activate
```

#### Linux / macOS

```bash
source venv/bin/activate
```

### 3. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 4. Настройка переменных окружения

Скопируйте файл:

```bash
.env_example
```

в

```bash
.env
```

и заполните его:

```env
DJANGO_SECRET_KEY=your_secret_key
DJANGO_DEBUG=True

POSTGRES_DB=team_finder_db
POSTGRES_USER=team_finder_user
POSTGRES_PASSWORD=your_password
POSTGRES_HOST=localhost
POSTGRES_PORT=5432

TASK_VERSION=1
```

### 5. Настройка PostgreSQL

Выполните команды:

```sql
CREATE DATABASE team_finder_db;

CREATE USER team_finder_user
WITH PASSWORD 'your_password';

GRANT ALL PRIVILEGES
ON DATABASE team_finder_db
TO team_finder_user;
```

### 6. Выполнение миграций

```bash
python manage.py makemigrations users projects
python manage.py migrate
```

### 7. Создание администратора

```bash
python manage.py createsuperuser
```

### 8. Запуск сервера

```bash
python manage.py runserver
```

После запуска приложение будет доступно по адресу:

```text
http://127.0.0.1:8000/
```

## Маршруты приложения

| Раздел | URL |
|----------|----------|
| Главная страница | `/projects/list/` |
| Регистрация | `/users/register/` |
| Вход | `/users/login/` |
| Профиль пользователя | `/users/<id>/` |
| Редактирование профиля | `/users/edit-profile/` |
| Смена пароля | `/users/change-password/` |
| Список пользователей | `/users/list/` |
| Избранные проекты | `/projects/favorites/` |
| Создание проекта | `/projects/create-project/` |
| Административная панель | `/admin/` |

> Фильтры на странице пользователей доступны только авторизованным пользователям.

## Возможные проблемы

Если приложение не запускается:

1. Убедитесь, что PostgreSQL запущен.
2. Проверьте правильность настроек в файле `.env`.
3. Убедитесь, что база данных создана.
4. Проверьте корректность порта PostgreSQL.
5. Выполните миграции повторно.

```bash
python manage.py migrate
```
